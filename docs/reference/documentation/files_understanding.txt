**app_files/**
* `build.gradle.kts`: The Gradle build script for the Android application module, defining plugins, dependencies, and build configurations.
* `google-services.json`: Configuration file containing Firebase project details and API keys for connecting the app to Firebase services.
* `Implementation Plan 25Apr.md`: Markdown file outlining the detailed implementation plan for version 1.4.
* `Loan Application Data Catalog 25Apr.md`: Markdown file containing the detailed data catalog for version 1.4.
* `Loan Application Requirements 25Apr.md`: Markdown file detailing the requirements and technical specifications for version 1.4.
* `strings.xml`: Android resources file containing string literals used throughout the application.
* `file_provider.xml`: XML configuration defining accessible file paths for the `FileProvider`.
* `AndroidManifest.xml`: The core configuration file for the Android application, defining permissions, components, and providers.

**com/loansai/unassisted/**
* `MainApplication.kt`: Custom `Application` class used for Hilt initialization and other app-level setup like Timber and Firebase.
* `MainActivity.kt`: The main entry point activity for the Android application.

**com/loansai/unassisted/data/local/database/di/**

**com/loansai/unassisted/data/model/**

* `EmploymentDetailsDto.kt`: Defines Data Transfer Objects (DTOs) for employment details (`EmploymentDetailsDto`, `EmployerDto`).
* `LoanApplicationDto.kt`: Defines Data Transfer Objects (DTOs) for `LoanApplication` and `LoanOffer`.
* `PANDetailsDto.kt`: Defines Data Transfer Objects (DTOs) for PAN details (`PANDetailsDto`) and bureau reports (`BureauReportDto`).
* `PersonalInfoDto.kt`: Defines Data Transfer Objects (DTOs) for personal information (`PersonalInfoDto`, `AddressDto`).
* `UserDto.kt`: Defines Data Transfer Objects (DTOs) for user data and authentication requests/responses.

**com/loansai/unassisted/data/remote/api/**

* `DocumentApi.kt`: Retrofit interface for document-related API calls (upload, process, get, delete).

* `LoanApi.kt`: Retrofit interface for loan application-related API calls (get, update, submit, calculate offer, etc.).

**com/loansai/unassisted/domain/model/**
* `AIAssistantMessage.kt`: Defines data classes for AI assistant messages and conversations (`AIAssistantMessage`, `AIConversation`, `MessageType`).
* `ApplicationStep.kt`: Defines an enum (`ApplicationStep`) representing the different steps in the loan application UI flow.
* `AppMetadata.kt`: (Deprecated) Defines data classes for application metadata; notes indicate these are now in `MetadataModels.kt`.
* `BREModels.kt`: Defines data classes related to the Business Rules Engine (BRE) input and output (`BREInput`, `BREOutput`, `BureauData`, etc.).
* `BureauModels.kt`: Defines data classes related to credit bureau information (`BureauAddress`, `BureauAccountSummary`).
* `BureauReport.kt`: Defines the main data class for the credit Bureau Report (`BureauReport`) and related enums/models.
* `Document.kt`: Defines domain models related to documents (`Document`, `DocumentProcessingResult`, `BankStatementData`, etc.) and associated enums (`DocumentType`, `FileType`, etc.).
* `EmploymentDetails.kt`: Defines domain models for employment details (`EmploymentDetails`, `Employer`) and related enums (`EmploymentType`, `EmployerCategory`, `VerificationMethod`).
* `LoanApplication.kt`: Defines the core domain model for a `LoanApplication` and related enums (`ApplicationStatus`, `DecisionStatus`, `OfferStatus`).
* `LoanOffer.kt`: Defines a UI-specific data class (`LoanOfferUI`) for displaying loan offers.
* `LoanOfferStatus.kt`: Defines an enum (`LoanOfferStatus`) representing different statuses of a loan offer (likely deprecated or replaced by `OfferStatus` in `LoanApplication.kt`).
* `MetadataModels.kt`: Defines domain models for various metadata types (`ScreenVisit`, `SectionTiming`, `DocumentEvent`, etc.) and the `MetadataRepository` interface.
* `OCRScanState.kt`: Defines a sealed class (`OCRScanState`) representing the different states of an OCR scanning process.
* `PANDetails.kt`: Defines domain models for PAN details (`PANDetails`) and payment history (`PaymentHistory`).
* `PersonalInfo.kt`: Defines domain models for personal information (`PersonalInfo`, `Address`) and related enums (`MaritalStatus`, `AddressType`).
* `User.kt`: Defines the domain model for a `User`.
